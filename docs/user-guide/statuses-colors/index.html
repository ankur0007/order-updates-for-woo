Statuses & colours

Statuses are short colour-coded labels on every update card. They sort the conversation visually and power the orders-list filter, but they don’t gate any plugin behaviour.

What a status is

Each update has a single status — a short label with a colour. It appears on the update card as a pill, and the colour also stripes down the left edge of the card and the customer’s thread bubbles.

Status is purely organisational — it doesn’t gate any feature. Change it any time; the change is logged and shown as a system marker in the customer thread.

Default options

  • Neutral — grey.
  • Notice — blue.
  • Warning — amber.
  • Resolved — green. Auto-applied on solve.

Customising statuses

Under Settings → General → Statuses, rename existing options, change colours, add new ones, remove unused ones. One status is always the “solved” sentinel — auto-applied when an update is closed.

Filtering by status on the orders list

The most-recent update status appears in the Updates column on the WooCommerce orders list. Combined with the “Has unsolved updates” filter, this gives you a triage view across the whole store.
